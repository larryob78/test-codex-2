# Starter Pack Wiring Checklist

This README is your wiring guide for connecting the AgentKit configuration, evals, and graders to your production workflow.

## Day 0 Setup

1. Import `agentkit_config.json` into Agent Builder (or replicate manually if the import schema differs). Use `agentkit build` (or `python -m agentkit.cli build`) to materialise the manifest bundle if you prefer a folder + zip instead of the raw JSON.
2. Enable the `web_search`, `file_search`, and `computer_use` built-in tools inside your Responses API project settings.
3. Confirm access to models `o3`, `o4-mini`, `gpt-5`, and `gpt-5-pro`; request GPT-5 RFT beta if eligible.
4. Register connectors for Google Drive, SharePoint, and Teams in Connector Registry with read-only scopes.
5. Upload house brand assets to your file search vector store and tag them with consistent metadata (`brand`, `asset_type`, `compliance`).

## 30-Day Plan

* Stand up ChatKit for internal reviewers; route orchestrator transcripts into your logging stack.
* Run the 12-sample eval dataset with the proposed graders. Iterate prompts until you achieve \>=90% pass rate on tone and claims.
* Integrate Guardrails templates for PII masking and jailbreak detection.
* Pilot the Realtime SIP flow with the creative director to validate live approval loops.

## 60-Day Plan

* Expand the eval dataset to 25–50 samples per asset class (email, paid social, video scripts).
* Add Prompt Caching for long-form brand canon prompts and enable Predicted Outputs on templated deliverables.
* Layer in tool-use grading to verify that agents prefer file search over web search when on-brand data exists.
* Connect the Producer agent’s computer-use logs to your compliance archive.

## 90-Day Plan

* Enable automated Sora renders for select storyboard beats and feed resulting clips back into the asset library.
* Introduce A/B testing variants generated by the Copywriter batch mode and push metrics into your experimentation platform.
* Apply custom guardrails for regional legal requirements (GDPR, LGPD) and document overrides in the BrandCard.
* Prepare for Agent Builder GA import/export by tracking any manual schema adjustments in this README.

## Operational Notes

* Treat `agentkit_config.json` as the single source of truth for prompts and schemas.
* Update `house_style_graders.proposed.json` when your house style evolves; version the file alongside prompt changes.
* Replace placeholder citations within the Sources section of the main `README.md` once you add official documentation URLs.
